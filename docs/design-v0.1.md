\# RepoFit Packet Design v0.1



\## 1. 프로젝트 정의



RepoFit Packet은 프론트엔드 취업 지원 전에 채용공고, GitHub 저장소, 첨부 자료, 프로젝트 메모, 보완 작업을 하나의 지원 패킷으로 묶어 제출 전 점검 리포트를 만드는 도구다.



\## 2. 핵심 문제



취업 지원 과정에서 공고 내용, 포트폴리오 프로젝트, 이력서/PPT 파일, 보완 메모가 흩어져 있어 제출 전에 무엇이 부족한지 확인하기 어렵다.



기존 RepoFit은 GitHub 저장소를 불러와 포트폴리오 후보를 정리하는 데 집중했다.  

RepoFit Packet은 한 단계 더 나아가 공고 하나를 기준으로 어떤 프로젝트를 제출할지, 어떤 증거가 있는지, 무엇을 보완해야 하는지를 정리한다.



\## 3. 핵심 사용자



1차 사용자는 프론트엔드 취업을 준비하는 박진호 본인이다.  

이후 비슷하게 포트폴리오 기반으로 지원하는 신입 개발자까지 확장할 수 있다.



\## 4. 핵심 사용 흐름



1\. 사용자가 회사명, 포지션, 채용공고 내용을 입력한다.

2\. 사용자가 제출 후보 GitHub 저장소 URL을 입력한다.

3\. 앱이 GitHub 저장소의 기본 정보, README, package.json 정보를 가져온다.

4\. 앱이 채용공고에서 요구 기술 키워드를 추출한다.

5\. 앱이 공고 요구 기술과 GitHub 저장소에서 확인된 증거를 비교한다.

6\. 앱이 부족한 항목을 체크리스트와 보완 과제로 생성한다.

7\. 사용자가 첨부 자료 링크와 메모를 추가한다.

8\. 앱이 제출 전 점검 리포트를 미리보기로 보여준다.

9\. 사용자가 이메일 요약을 전송한다.



\## 5. v0.1 MVP 범위



\- 지원 패킷 생성

\- 회사명 / 포지션 / 공고 텍스트 입력

\- GitHub public repo URL 입력

\- repo 기본 정보 분석

\- README 존재 여부 확인

\- package.json 의존성 확인

\- 공고 기술 키워드 추출

\- 기술 요구사항과 repo 증거 매칭

\- 체크리스트 생성

\- 보완 과제 생성

\- 첨부 링크 / 파일 메타데이터 등록

\- 리포트 미리보기

\- 이메일 요약 전송



\## 6. v0.1에서 제외할 기능



\- GitHub OAuth

\- private repo 분석

\- GitHub GraphQL

\- PR / Issue 깊은 분석

\- PDF / PPT 내용 자동 분석

\- AI 자기소개서 작성

\- 채용 사이트 자동 연동

\- Gmail 분석

\- 자동 지원

\- 팀 협업 기능

\- 이메일 웹훅

\- 복잡한 대시보드 차트



\## 7. 자동화 경계



자동화할 것:



\- GitHub 저장소 기본 정보 수집

\- README / package.json 확인

\- 공고 기술 키워드 추출

\- 체크리스트 생성

\- 보완 과제 생성

\- 이메일 요약 초안 생성



사용자가 확인해야 할 것:



\- 실제로 본인이 구현한 내용인지

\- 프로젝트 설명이 사실과 맞는지

\- 최종 보완 과제를 수행했는지

\- 이메일을 실제로 전송할지



\## 8. v0.1 성공 기준



많은 기능을 넣는 것이 아니라, 공고 하나를 기준으로 다음 흐름을 끝까지 완성하는 것이 성공 기준이다.



공고 입력 → repo 분석 → 기술 매칭 → 보완 과제 생성 → 리포트 미리보기 → 이메일 요약 전송
